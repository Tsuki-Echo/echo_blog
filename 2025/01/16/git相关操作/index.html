Git常用命令速查表

功能 命令 比喻
回退版本 git reset –参数 [提交的版本id] 用法在下面
查看工作区文件 ls
查看暂存区文件 git ls-files
查看本地版本库文件 git ls-tree 列出指定提交或分支中的文件和文件夹。
查看提交历史 git log --oneline 以一行的简洁方式展示提交历史，每个提交记录占据一行，适合快速浏览提交历史的场景。

获取远程仓库的修改（变化） git fetch

git三区

本地 –工作区

add –暂存区

commit –版本库

HEAD

它指向分支的最新提交节点

git文件状态

Untrack

指刚创建未被git管理的文件

Unmodified

指被git管理但为修改过的文件

Modified

指被git管理并且已修改但是未被添加到暂存区的文件

Staged

指已经修改后并添加到暂存区里的文件

git reset

用于日常中需要回退到之前的某一个版本的情形下，回退到之前某一个版本。

下面三个例子都是在提交了三次commit的情况下分开执行，commit有序1，2，3。

git reset –soft [提交的版本id]

让其回退到commit2，版本指向commit2，commit3记录消失，但是工作区和暂存区的文件没变，即失忆了，忘记后面的事，但做的事（文件）还在。

git reset –hard [提交的版本id]

让其回退到commit2，真正的让其版本指向commit2的时候，工作区和暂存区在commit2后面的内容都被清空。

git reset (–mixed) [提交的版本id]

让其回退到commit2，版本指向commit2，工作区的关于commit2后面的还在，但是暂存区的已经清空。

git diff

git diff (不加参数)

比较工作区和暂存区之间的差异，如下

第一行表示被修改的文件，第二行后面10064是文件权限，红色表示删除内容，绿色表示新增的内容。

git diff HEAD

比较工作区和版本库之间的差异

git diff –cached

比较暂存区和版本库之间的差异

git diff [提交的版本id1] [提交的版本id2]

比较两个版本之间的差异

git diff [提交的版本id] HEAD

比较某个版本和当前最新版本之间的差异

git diff HEAD~[数字] HEAD

比较上一(数字代表前几个版本)个版本和当前最新版本之间的差异

还可以在后面加[文件名]表示只比较某个文件的差异内容

也可以查看两个分支之间的差异
